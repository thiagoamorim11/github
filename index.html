Está com dificuldades nas plataformas Git, GitHub ou GitPages? Aprenda agora mesmo com nosso tutorial.

PASSO 1 - Download

Primeiro de tudo precisamos fazer o download do Git em seu site oficial (DOWNLOAD)

PASSO 2 - GitHub Repositórios

Agora no GitHub com sua conta logada você deve criar um repositório e escolher se o mesmo será privado ou público, logo após isso, precisaremos do link que aparece na aba "code"

PASSO 3 - Git Config

Com o Git instalado, podemos utilizar a ferramenta Git bash ou o prompt de comando (nesse tutorial utilizarei o cmd, mas sinta-se livre para usar o git bash). Vamos abrir o cmd(Tecla windows + r -> Digite "cmd" + enter) e configurar o nome de usuário (git config --global user.name "nome") e email (git config --global user.email "email")

PASSO 4 - Git Clone

Utilizaremos o git clone após criar o repositório no GitHub, com o repositório criado copiaremos o link que aparece na aba x e colaremos no cmd após o comando "git clone (link)"

PASSO 5 - Code .

O comando "code ." é utilizado para abrir o visual studio code, caso não o tenha instalado, crie um arquivo txt chamado index e troque a extensão .txt por .hmtl e você poderá editá-lo com o bloco de notas.

PASSO 6 - Git add

O comando Git add adiciona arquivos novos ou modificados em seu diretório

PASSO 7 - Git Commit

Com o arquivo index.html adicionado vamos utilizar o git commit como um "ponto de salvamento" onde com vários commits iremos criar a histório da aplicação.

PASSO 8 - Git Push

Com os commits devidamente feitos é hora de colocar os arquivos locais no GitHub, para isso usaremos o git push, porém, perceba que acontecerá um erro aqui. Para sairmos desse erro temos duas opções

Outros comandos

Git Init Git Pull Git Branch Git Checkout Git Merge Git Status

Git Init

O Git init é usado para criar a pasta .git em um repositório de sua escolha, só será preciso ficar atento se o mesmo repositório está presente no GitHub para que não haja erro na hora de usar o git push

Git Pull

Com o git push nós "empurramos" nosso código do local para o GitHub. Já o git pull será responsável por pegar do GitHub para a máquina local um arquivo que foi edital por outra pessoa

Git Branch

Por padrão, sempre que iniciamos o git, utilizamos ele no "ramo" master, caso você esteja com medo de causar um erro no seu código master, você pode criar outro ramo com o comando git branch "nome da branch"

Git Checkout

Se você reparou no print anterior, quando demos o git status, ele continua na branch master, para trocar para a nova branch utilizaremos o comando git checkout (nome da branch)

Git Merge

Não podemos ficar com duas branchs para sempre, eventualmente teremos que voltar para branch master, mas como todas as atualizações estão na branch versao2, precisamos voltar para a master, e juntar as duas branchs

Git Status

Por fim, utilizamos o git status para mostrar o que está acontecendo com o nosso git, caso haja itens a serem commitados ele nos avisará
